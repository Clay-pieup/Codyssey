# 산출물 1. LLM 모델 비교·선정 보고서

## 1. 문서 개요

- **프로젝트명**: 리조트 비수기 활성화 보고서 개요 생성 자동화
- **문서 목적**: 한산마리나리조트의 비수기 활성화 전략 보고서 개요를 생성하기 위한 최적 LLM을 비교·선정한다.
- **작성 대상**: 내부 의사결정자, 프롬프트 설계자, 운영 담당자
- **버전**: v1.1

---

## 2. 과업 정의

### 2.1 해결하려는 업무 문제
한산마리나리조트는 보유 자산(31실 객실, 요트 4~6척, 자체 마리나 시설)을 갖고 있음에도 비수기 객실 가동률이 약 30% 수준에 머물러 있다.  
내부 마케팅 담당자는 비수기 활성화 아이디어를 빠르게 구조화한 뒤, 임원 보고용 전략 개요로 정리해야 한다.  
하지만 매번 시장 맥락, 자사 자산, 실행 아이디어를 처음부터 조합해 초안을 만드는 데 시간이 많이 든다.

### 2.2 자동화 목표
LLM을 활용해 다음 조건을 만족하는 **전략 보고서 개요 초안**을 안정적으로 생성하도록 한다.

- 자사 맥락을 반영할 것
- 단순 아이디어 나열이 아니라 논리적 구조를 가질 것
- 실행 가능성이 높은 제안을 포함할 것
- 검증이 필요한 주장과 확정 가능한 사실을 구분할 것
- 임원 보고용으로 활용 가능한 톤과 형식을 갖출 것

---

## 3. 사용자 및 페르소나 설정

### 3.1 타겟 사용자
- **중견 인하우스 마케터**
- 리조트/숙박업 브랜딩 또는 퍼포먼스 마케팅 경험 보유
- 대표/임원에게 전략 방향을 보고해야 함
- 시간이 부족해 빠른 초안 생성과 논리 구조화가 중요함

### 3.2 LLM이 수행해야 할 페르소나
- **리조트 마케팅 전문 데이터 애널리스트**
- 단순 카피라이터가 아니라,
  - 수요 원인을 구조화하고
  - 자사 자산을 사업 관점에서 재해석하며
  - 실행 가능한 패키지/로드맵/KPI까지 제안하는 역할을 수행해야 함

## 4. 평가 대상 모델 및 실험 조건

### 4.1 비교 모델
1. **Gemini Pro 3.1**
2. **GPT-5.4**
3. **Claude Sonnet 4.6**

### 4.2 입력 방식
- 동일한 **웹 채팅 입력 방식** 사용
- 동일한 시스템 프롬프트 및 사용자 입력 템플릿 적용
- 동일 과업에 대해 각 모델 1회 응답 수집
- 응답은 보고서 개요 품질 중심으로 비교

### 4.3 테스트 입력 템플릿
다음과 같은 맥락을 공통 입력으로 사용했다.

- 대상: **한산마리나리조트**
- 보유 객실: **31실**
- 보유 자산: **요트 4~6척**, 자체 마리나 시설
- 비수기 객실 가동률: **약 30%**
- 과업: 비수기 수요를 높이기 위한 전략 보고서 개요 작성
- 출력 요구: 구조화된 챕터, 인사이트, 실행 제안, 검증 포인트 포함

### 4.4 공통 테스트 프롬프트 전문

#### 시스템 프롬프트

```text
============================
### SYSTEM INSTRUCTIONS ###
============================

# 역할
당신은 리조트 마케팅 전문 데이터 애널리스트입니다.
국내 중소형 리조트의 비수기 활성화 전략 수립을 10년 이상 담당해왔으며,
데이터 기반의 실행 가능한 보고서 개요를 작성하는 것이 전문 분야입니다.

# 대상 사용자
중견 인하우스 마케터 (경력 5~7년차)
임원 보고용 자료를 준비 중이며, 데이터 기반의 전문적 톤을 선호합니다.

# 핵심 원칙
1. [질문 우선] 자사 정보가 부족하면 개요 작성 전에 먼저 확인 질문을 하세요.
2. [환각 억제] 확실하지 않은 수치·사례·통계는 반드시 "확인 필요"로 표시하세요.
3. [자사 맥락 반영] 사용자가 제공한 자사 자산·환경을 반드시 전략에 녹여내세요.
4. [실행 가능성] 일반론이 아닌, 해당 리조트 규모·자원에 맞는 구체적 제안을 하세요.
5. [톤 유지] 임원 보고용의 간결·전문·데이터 기반 톤을 유지하세요.

# 출력 형식
보고서 개요는 반드시 아래 6개 챕터 구조로 작성합니다.
각 챕터마다 다음 4개 요소를 포함하세요:
- 제목
- 핵심 메시지 (1~2문장)
- 세부 논점 (3~5개 불릿)
- 근거 (데이터 출처 또는 "확인 필요" 표시)

[6개 챕터 구조]
1) 현황 진단
2) 시장 및 경쟁사 분석
3) 타겟 세분화
4) 핵심 전략 방향
5) 실행 방안
6) 기대 효과 및 KPI

# 마지막에 반드시 포함할 것
- ⚠️ 환각 검증 필요 항목 리스트 (사용자가 별도 확인해야 할 데이터·사례)

# 금지사항
- 근거 없는 구체적 수치 제시 (예: "매출 30% 증가 예상" 같은 무근거 표현)
- "SNS 강화", "고객 만족도 제고" 등 일반론적 표현
- 자사 자산(요트·마리나 등)을 반영하지 않은 범용 제안
```

#### 사용자 프롬프트

```text
============================
### USER REQUEST ###
============================

# 과업
우리 리조트의 비수기(11월~4월) 활성화 방안 보고서 개요를 작성해주세요.
다음 주 임원 보고에 사용할 예정입니다.

# 자사 정보
- 리조트명: 한산마리나리조트
- 위치: 경상남도 통영시
- 규모: 객실 31실 (소규모 부티크 리조트)
- 특수 자산: 요트 4~6척 보유, 자체 마리나 시설 운영
- 현황: 
  · 성수기(5~10월) 가동률 약 70%
  · 비수기(11~4월) 가동률 약 30%
  · 성수기/비수기 격차 약 40%p
- 타겟 고객: 현재는 가족 단위 여행객 중심 (여름 성수기)

# 원하는 결과물
- 6개 챕터 구조의 보고서 개요
- 각 챕터별 핵심 메시지 + 세부 논점 + 근거
- 요트·마리나 등 자사 고유 자산 활용 방안 반드시 포함
- 임원 보고용의 간결하고 전문적인 톤

# 참고사항
추가로 확인이 필요한 정보가 있다면 개요 작성 전에 먼저 질문해주세요.
```

## 7. 모델별 평가 요약

### 7.1 Gemini Pro 3.1

#### 강점
- 6개 챕터 구조를 안정적으로 준수했다.
- 요트·마리나 자산을 전반적으로 잘 활용했다.
- “고요함의 재정의”와 같은 브랜딩형 프레이밍이 돋보였다.
- 전반적으로 창의적인 아이디어가 풍부했다.

#### 한계
- 검증 포인트가 응답 내에 흩어져 있어 실무 검토 편의성이 다소 떨어졌다.
- 실행 로드맵이나 KPI 수준의 구체성이 상대적으로 부족했다.
- 표현이 다소 브레인스토밍 톤에 가까워 임원 보고 문체로는 약간 덜 정제됐다.

#### 총평
창의적 방향성은 좋았지만, **실행·검증 프레임까지 한 번에 가져가야 하는 업무용 출력물**로는 약간의 후처리가 필요했다.

---

### 7.2 GPT-5.4

#### 강점
- 구조와 논리 전개가 매우 정제되어 있었다.
- “비수기 = 방문 명분 부재 시즌”이라는 문제 재정의가 가장 선명했다.
- 검증 필요 항목을 별도 리스트로 정리해 환각 억제 측면에서 가장 우수했다.
- 임원 보고용 톤과 문서 완성도가 높았다.

#### 한계
- 자사 수치와 현장 맥락을 파고드는 정도는 Claude보다 약했다.
- 방향성은 좋지만 구체적인 패키지 설계, 단계별 실행안, KPI 연결은 다소 약했다.

#### 총평
**안전하고 정제된 전략 문서 초안**이 필요할 때 매우 강력한 선택지였다.  
다만 “현장 실행 설계”까지 바로 연결되는 깊이는 1위 모델보다 약간 부족했다.

---

### 7.3 Claude Sonnet 4.6

#### 강점
- 자사 수치와 운영 현실을 가장 적극적으로 반영했다.
- “31실 × 21실 미판매”처럼 문제를 정량적으로 해석해 실무 감각이 좋았다.
- 요트를 단순 어메니티가 아니라 독립 수익 상품으로 재해석한 점이 인상적이었다.
- 패키지, Phase 로드맵, KPI 표, 협업 방향까지 포함해 실행 가능성이 가장 높았다.

#### 한계
- 검증 리스트는 우수했지만 일부 벤치마크성 표현은 별도 확인이 필요했다.
- 가독성을 위해 사용한 이모지가 다소 많아, 가장 정제된 임원 톤이라고 보긴 어려웠다.

#### 총평
이번 과업에서 가장 중요한 기준이었던 **자사 맞춤성 + 실행 가능성**을 가장 잘 만족했다.  
후속 편집만 거치면 실무 활용도가 가장 높다고 판단했다.

## 8. 최종 선정 결과

### 8.1 선정 모델
**Claude Sonnet 4.6**

### 8.2 선정 이유
Claude Sonnet 4.6을 최종 모델로 선정한 이유는 다음과 같다.

1. **자사 맥락 반영력이 가장 높음**
   - 객실 수, 비수기 가동률, 요트 자산 등 핵심 정보를 실질적으로 활용했다.

2. **실행 가능한 제안 수준이 가장 높음**
   - 단순 아이디어가 아니라 패키지, 단계별 추진안, KPI까지 제시했다.

3. **전략 보고서 개요 목적에 가장 적합함**
   - 보고서 목차와 논리 흐름이 안정적이며 실무 후속 작업으로 연결하기 쉬웠다.

4. **내부 마케터의 시간 절감 효과가 가장 큼**
   - 후편집만 거치면 회의 자료, 제안서, 보고서 초안으로 바로 전환 가능했다.

### 8.3 운영 시 유의점
최종 선정과 별개로 아래 항목은 운영 단계에서 보완이 필요하다.

- 외부 벤치마크, 업계 사례, 수요 수치 관련 표현은 **반드시 재검증**
- 과장 표현을 줄이기 위해 **검증 체크리스트를 시스템 차원에서 강제**
- 이모지/장식 표현은 줄이고 **임원 보고용 서술 톤으로 후처리** 필요

## 9. 운영 권고안

최종 모델을 Claude Sonnet 4.6으로 사용하되, 다음과 같은 운영 방식을 권장한다.

### 9.1 기본 운영 모델
- **1차 생성**: Claude Sonnet 4.6
- **후편집 기준**
  - 검증 불가 표현 삭제
  - 이모지 제거 또는 축소
  - KPI 및 일정 표현 표준화
  - 최종 보고서 문체 통일

### 9.2 품질 안정화 장치
- 시스템 프롬프트에 다음 요구를 명시
  - 검증 불가 사실은 단정하지 말 것
  - 자사 입력값을 최소 3회 이상 명시적으로 활용할 것
  - 실행 항목마다 KPI 또는 기대 효과를 연결할 것
  - 벤치마크는 “예시 가능성” 수준으로만 언급할 것

### 9.3 향후 보완 방향
- Few-shot 예시를 추가해 톤을 더 정제
- 검증용 체크리스트 출력을 필수화
- 보고서 개요뿐 아니라 **캠페인 기획안 / 패키지 제안서 / 월간 실행안**까지 확장 검토

## 10. 결론

세 모델 모두 높은 수준의 응답을 생성했으며, 큰 점수 차이는 없었다.  
그러나 본 과업의 핵심은 단순한 문장 품질이 아니라 **자사 맥락을 반영한 실행 가능한 전략 개요**를 빠르게 만드는 데 있다.

이 기준에서 볼 때:

- **Gemini Pro 3.1**은 창의성이 강점이었고,
- **GPT-5.4**는 가장 정제된 보고서 톤과 검증 안정성을 보였으며,

- 보고용 문체 통일을 위해 최종 편집 단계에서 표현을 정제할 것
